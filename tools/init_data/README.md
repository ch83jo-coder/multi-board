# 初期データ投入ツール

OpenAIで日本語の投稿とコメントを生成し、Supabaseの有効な各掲示板へ投入する手動ツールです。投稿者、コメント投稿者、投票者には8名のシード用ボットを使用します。

## 投入されるデータ

- `is_active = true`の掲示板ごとに投稿10件
- 投稿ごとにコメント2〜5件（返信を含む）
- 投稿者以外のボットによるランダムな投票
- 50〜2,000のランダムな閲覧数
- 過去7日以内の投稿日時と、それ以降のコメント日時

投稿数、コメント数、投票数はDBトリガーで更新します。

## 事前準備

Node.js 22を推奨します。Supabaseのマイグレーションを適用したうえで、`.env.local`に次の値を設定してください。

```dotenv
NEXT_PUBLIC_SUPABASE_URL=https://<project-ref>.supabase.co
SUPABASE_SERVICE_ROLE_KEY=<service-role-key>
OPENAI_API_KEY=<openai-api-key>
OPENAI_MODEL=gpt-4o-mini
```

`OPENAI_MODEL`は省略可能で、既定値は`gpt-4o-mini`です。`SUPABASE_SERVICE_ROLE_KEY`と`OPENAI_API_KEY`は秘密鍵です。ブラウザへ公開したり、Gitへコミットしたりしないでください。

## 実行方法

```bash
yarn seed
```

既に投稿が1件以上ある掲示板はスキップされます。同じコマンドを再実行しても投稿は重複しません。既存投稿の有無にかかわらず追加したい場合だけ、`--force`を指定します。

```bash
yarn seed --force
```

`--force`は既存データを削除せず、投稿をさらに10件追加します。

## 動作と費用

有効な掲示板ごとにOpenAI Chat Completions APIを1回呼び出します。料金と処理時間は掲示板数、モデル、生成量によって変わるため、実行結果に表示されるトークン数とOpenAIの利用状況を確認してください。

ボットはユーザー名とメールアドレスを基準に再利用されます。掲示板への投入途中で失敗した場合、その実行中に同じ掲示板へ追加した投稿は関連コメント・投票とともに削除されます。

## 実行結果

完了時に掲示板ごとの投稿数、コメント数、投票数と所要時間を表示します。環境変数不足、OpenAI APIエラー、Supabaseエラーが発生した場合は、処理段階を含むエラーメッセージを表示して終了コード`1`で終了します。
